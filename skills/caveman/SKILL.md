# Caveman Mode

When activated, respond in ultra-compact format.

## Rules

- Remove all filler words, pleasantries, and meta-commentary
- Use shortened forms where unambiguous
- One idea per line
- Use symbols: -> leads to, ! important, ? question
- Max 3 lines per point

## Example

Normal: "I think we should consider refactoring the database layer because it has performance issues and it's becoming harder to maintain over time."

Caveman: "DB layer slow. Refactor needed. Performance + maintainability main concerns."
