# 周期性自检与违例恢复

## 触发时机

每完成 **2 个子任务**，或每 **5 分钟**无输出，必须执行一次自检。

## 自检内容

```
[自检] Phase: {X} | 协议遵循度检查
- 最近 3 条消息是否包含 [Auto] 前缀？ [是/否]
- 当前 Phase 门禁是否全部通过？ [是/否]
- 是否有未派发的 Level 0 任务？ [是/否]
- 最近 3 次诊断尝试是否用了不同策略？ [是/否]
- 心超过期？ [是/否]（上次输出 > 5 分钟前）
- MANDATORY 脚本当前执行到 STEP: {N}
```

## 违例分级与处理

| 级别 | 条件 | 处理 |
|------|------|------|
| L1 | 缺少 `[Auto]` 前缀 | 立即纠正，后续消息使用正确格式 |
| L2 | Phase 门禁未全部通过 | 回到对应 Phase 重新执行门禁清单 |
| L2 | 有未派发的 Level 0 任务 | 立即通过 task() 并行派发 |
| L2 | 心超过期 | 输出心跳后继续 |
| L3 | 同一策略连续失败 ≥3 次 | 强制换方法（切换技能/思路） |
| L3 | 连续 2 次自检发现同一问题 | 记录到 `error_patterns.json`（confidence+0.1） |
| L4 | 连续 3 次自检全部偏离 | 输出 `[协议违规] 无法自主恢复`，通知用户 |

## 协议偏离恢复流程

```
检测到偏离
   │
   ▼
输出 [协议违规] 检测到偏离: {具体缺失项}
   │
   ▼
确认当前 Phase 和 MANDATORY STEP
   │
   ▼
回到对应 STEP 重新执行
   │
   ▼
恢复后执行一次完整自检
   │
   ▼
全部通过 → 继续 | 仍有偏离 → 升级级别
```

## 心跳机制

长时间无输出时，必须主动发送心跳：

```
[心跳] (N/M) 进行中: 任务列表 | Phase: {X}
[心跳] 自检: 协议正常? [是/否] | 门禁通过? [是/否]
```

## 违例学习（self-improving 集成）

执行完整任务后，将自检发现的违例记录到学习系统：

```
[ProtocolViolation] {Y-m-d H:M} | Phase {X} | {缺失项} | {上下文摘要}
```

如果同一类型违例在同一个 Phase 出现 ≥2 次：
- 对应 MANDATORY STEP 前自动添加 `⚠️ 历史违例高发步骤，必须严格执行`
- `error_patterns.json` 中该模式 confidence +0.1
