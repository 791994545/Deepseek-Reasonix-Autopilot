# 子代理并行执行协议

> **Reasonix 适配**: 此规则源自 OpenCode，已在 Reasonix 平台生效。工具名已映射（Read→read_file, Bash→run_command 等），行为规则保持不变。

## 何时派发给子代理
必须用 task()/子代理的场景：
- **多个独立子任务**（无交叉依赖）→ 并行派发
- **探索与执行可分离** → 探索提前发
- **任务类型匹配专业代理**（如 explore 用于代码理解）
- **当前会话上下文不足** → 卸载任务到隔离 agent

禁止派发：
- **状态决策操作**（路由/冲突消解/Gate 决策）
- **依赖当前上下文完成情况的子任务**
- **子代理没权限用的工具**

## 派发模式
1. **分类子任务**：只读？写？命令？网络？
2. **映射到最佳代理类型**
3. 同类任务可并行，异类按类型拆分

## 执行
- 每个子代理收到：自包含 prompt（上下文摘要 + 任务定义 + 输出格式）
- 子代理不能再生子代理（depth=1）
- 设置合适的超时时间
- 收集结果

## 结果聚合
- 收集全部结果后再进入下一 Gate
- 有子代理失败 → 分析失败原因：重试/跳过/阻塞
- 失败的子代理不取消其他正在运行的子代理
- 2+ 子代理同样错误 → 可能是系统性 → 暂停并行模式
