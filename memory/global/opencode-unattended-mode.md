# 无人值守模式规则

> **Reasonix 适配**: 此规则源自 OpenCode，已在 Reasonix 平台生效。工具名已映射（Read→read_file, Bash→run_command 等），行为规则保持不变。

## 激活条件
用户明确说"自动"/"无人值守"/"别问我"/"直接执行"/"通宵跑"；或命令含 `--auto`/`--unattended`

## 核心原则
**目标是完成任务，不是跳过任务。** 降级/简化/跳过子任务 = 逃避，不是无人值守。

## 失败处理
子任务执行 → 成功则继续 → 失败则**换策略**重试，不设上限。
- 方法 A 失败 → 试方法 B（完全不同思路）
- 方法 B 失败 → 试方法 C（查文档/搜索/替代方案）
- 唯一放弃条件：外部依赖确认不可用（DB断开/API挂了/网络不通）

## 绝对不能做
❌ "这个子任务失败 3 次了，跳过吧"
❌ "完整功能做不了，做个简化版交差"
❌ 任何形式的"降低需求标准来完成任务"

## 循环检测
- 同一策略连续失败 3 次 → 强制换策略
- 本质相同的方法换了马甲 → 判定为循环，强制彻底换思路
- 单子任务超过 20 分钟无进展 → 输出进度，重新评估
- 真正卡住了：记录需人工介入的点，继续执行其他子任务

## 输出格式
`[无人值守] ✅ (N/M) 任务完成 | 🔄 任务进行中 | ⚠️ 需人工处理`
`[心跳] (N/M) 进行中: xxx | 已耗时 Xm`
`[无人值守] 🏁 汇总: N完成 | 0跳过 | M需人工`
