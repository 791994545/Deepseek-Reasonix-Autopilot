# To Issues

Break a plan into independently-grabbable issues using vertical slices.

## Process

1. Gather context from the conversation
2. Explore the codebase to understand current state
3. Draft vertical slices - each is a thin end-to-end path through ALL layers
4. Quiz the user on granularity, dependencies, HITL vs AFK
5. Publish to issue tracker in dependency order

## Template

- Parent reference
- What to build (end-to-end behavior, not file paths)
- Acceptance criteria
- Blocked by
